# Custom FAIR Data Machine Build

This build package was generated by the FAIR Data Machine Builder.

> [!TIP]
> This directory (`custom-build/`) is overwritten every time you click "Generate Build Package" in the UI.
> To retain this specific configuration, **copy or rename** this directory (e.g., to `my-project-build/`) before generating a new one.

## Build & Test
To build the machine and verify the installation:
```bash
# 1. Build the image
docker build -t my-custom-machine .

# 2. Run smoke tests
./test-image.sh my-custom-machine
```

## Quick Start

### 1. Interactive Shell
To launch the machine and enter an interactive shell as the `dartfx` user with your current directory mapped as a workspace:
```bash
docker run --rm -it \
  -v "$(pwd)":/home/dartfx/workspace \
  -w /home/dartfx/workspace \
  my-custom-machine \
  su - dartfx
```

### 2. Persistence
To persist data across container restarts (e.g., databases and models), map volumes for these paths:

| Component | Container Path | Description |
| --- | --- | --- |
| **Workspace** | `/home/dartfx/workspace` | Your project files and data |
| **PostgreSQL+** | `/var/lib/postgresql/data` | Relational database data |

Example with persistence:
```bash
docker run --rm -it \
  -v "$(pwd)":/home/dartfx/workspace \
  -v "pgdata":/var/lib/postgresql/data \
  -v "ollama_models":/usr/share/ollama/.ollama \
  my-custom-machine \
  su - dartfx
```
